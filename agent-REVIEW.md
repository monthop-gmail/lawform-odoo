# AI Review — คู่มือตรวจสำนวนคดี

คู่มือสำหรับ AI ที่ทำหน้าที่ตรวจสอบเอกสารศาลที่สร้างโดย AI ทนายหรือมนุษย์
เพื่อให้แน่ใจว่าสำนวนครบถ้วน ถูกต้อง พร้อมยื่นศาล

## หลักการตรวจ

ตรวจ **6 หัวข้อ** ตามลำดับ:
1. ชุดเอกสารครบไหม
2. Placeholder ครบไหม
3. เนื้อหาอิสระร่างครบไหม
4. บัญชีพยานครบไหม
5. ข้อมูลคู่ความถูกต้องไหม
6. เลขไทยถูกไหม

## 1. ชุดเอกสารมาตรฐาน

### คดีแพ่ง — ฝ่ายโจทก์ (บังคับ)

| ฟอร์ม | ต้องมี | ตรวจอะไร |
|-------|-------|---------|
| แบบ ๙ ใบแต่งทนาย | บังคับ | ขอบเขตอำนาจ ต้องไม่เป็นจุด |
| แบบ ๑ หนังสือมอบอำนาจ | บังคับ | ขอบเขต "ในเรื่อง" ต้องไม่เป็นจุด + agent_id ต้องมี |
| แบบ ๔ คำฟ้อง | บังคับ | ต้องมี continuous_text อย่างน้อย 3 ข้อ |
| แบบ ๕ คำขอท้ายฟ้องแพ่ง | บังคับ | ต้องมีคำขออย่างน้อย 2 ข้อ |
| แบบ ๑๕ บัญชีพยาน | บังคับ | ต้องมี witness_item อย่างน้อย 2 รายการ |

### คดีแพ่ง — ฝ่ายจำเลย (บังคับ)

| ฟอร์ม | ต้องมี | ตรวจอะไร |
|-------|-------|---------|
| แบบ ๙ ใบแต่งทนาย | บังคับ | เหมือนโจทก์ |
| แบบ ๑๑ คำให้การจำเลย | บังคับ | ต้องมีเนื้อหาตอบทุกข้อในคำฟ้อง |
| แบบ ๑๕ บัญชีพยาน | บังคับ | ต้องมี witness_item |

### คดีอาญา — ฝ่ายโจทก์ (บังคับ)

| ฟอร์ม | ต้องมี | ตรวจอะไร |
|-------|-------|---------|
| แบบ ๙ ใบแต่งทนาย | บังคับ | เหมือนแพ่ง |
| แบบ ๔ คำฟ้อง | บังคับ | continuous_text + อ้างมาตรา |
| แบบ ๖ คำขอท้ายฟ้องอาญา | บังคับ | **ต้องมีบทมาตรา** + คำขอลงโทษ + ออกหมาย |
| แบบ ๑๕ บัญชีพยาน | บังคับ | เหมือนแพ่ง |

### ประกันตัว (ถ้ามี)

| ฟอร์ม | ตรวจอะไร |
|-------|---------|
| แบบ ๕๗ คำร้องขอปล่อยชั่วคราว | guarantor_id + bail_amount ต้องมี + เหตุผลไม่เป็นจุด |
| แบบ ๕๘ สัญญาประกัน | guarantor_id + bail_amount ต้องตรงกับ ๕๗ |

### อุทธรณ์/ฎีกา (ถ้ามี)

| ฟอร์ม | ตรวจอะไร |
|-------|---------|
| แบบ ๓๒ อุทธรณ์ | ต้องมี continuous_text คัดค้านคำพิพากษา |
| แบบ ๓๓ ท้ายอุทธรณ์ | ข้อมูลผู้เรียง |
| แบบ ๓๖ ฎีกา | ต้องมี continuous_text + อ้างคำพิพากษาอุทธรณ์ |
| แบบ ๓๗ ท้ายฎีกา | ข้อมูลผู้เรียง |

## 2. เกณฑ์ตรวจ Placeholder

### ระดับความสำคัญ

| ระดับ | placeholder | เหตุผล |
|-------|------------|--------|
| ❌ บังคับ | `%(plaintiff)s`, `%(defendant)s` | ชื่อคู่ความ ขาดไม่ได้ |
| ❌ บังคับ | `%(court)s` | ชื่อศาล |
| ❌ บังคับ | `%(date_long)s` | วันที่เอกสาร |
| ❌ บังคับ | `%(charge)s` (ในคำฟ้อง) | ข้อหา |
| ❌ บังคับ | `%(claim_amount)s` + `%(claim_amount_text)s` | จำนวนเงิน |
| ⚠️ สำคัญ | `%(black_case_thai)s` | เลขคดีดำ (ได้ตอนยื่นฟ้อง) |
| ⚠️ สำคัญ | `%(plaintiff_id_no)s` | เลขบัตรประชาชนโจทก์ |
| ⚠️ สำคัญ | `%(lawyer_license_no)s` | เลขใบอนุญาตทนาย |
| ⚠️ สำคัญ | `%(lawyer_phone)s`, `%(lawyer_address)s` | ติดต่อทนาย |
| 💡 ดีถ้ามี | `%(red_case_thai)s` | เลขคดีแดง (ได้ทีหลัง) |
| 💡 ดีถ้ามี | `%(plaintiff_email)s`, `%(plaintiff_fax)s` | ช่องทางติดต่อเพิ่ม |
| 💡 ดีถ้ามี | `%(plaintiff_age)s`, `%(plaintiff_occupation)s` | ข้อมูลส่วนตัวเพิ่ม |

### กฎตรวจ

- **placeholder ว่าง = ข้อมูลใน res.partner/legal.case ว่าง** → ต้องเติมที่ต้นทาง ไม่ใช่แก้ body_html
- **`%(red_case_thai)s` ว่างได้** — ศาลออกทีหลังหลังรับฟ้อง
- **`%(black_case_thai)s` ว่างได้ชั่วคราว** — ได้ตอนยื่นฟ้อง แต่ต้องเตือน

## 3. เกณฑ์ตรวจเนื้อหาอิสระ

### จุดที่ยอมรับได้ (ไม่ต้องแก้)

| pattern | ที่ไหน | เหตุผล |
|---------|-------|--------|
| `ลงชื่อ ................` | ท้ายทุกฟอร์ม | ช่องลงลายมือชื่อ |
| `( .................. )` | ใต้ลงชื่อ | ช่องพิมพ์ชื่อ |
| `คำสั่ง ................` | ใบแต่งทนาย | ศาลเป็นคนสั่ง |
| ที่อยู่ผู้เขียน/พิมพ์ | ท้ายแบบ ๕, ๖, ๓๓, ๓๗ | คนพิมพ์เอกสาร (คนละคนกับทนาย) |
| `สำเนา ... ฉบับ` | ท้ายแบบ ๕, ๖ | จำนวนสำเนา (กรอกตอนยื่น) |

### จุดที่ต้องแก้ (ห้ามปล่อย)

| pattern | ที่ไหน | ต้องร่าง |
|---------|-------|---------|
| `ให้มีอำนาจ *...` | แบบ ๙ | ขอบเขตอำนาจทนาย |
| `ในเรื่อง ...` | แบบ ๑ | ขอบเขตมอบอำนาจ |
| `๑. ... ๒. ... ๓. ...` | แบบ ๕, ๖ | คำขอบังคับ |
| `บทมาตรา ...` | แบบ ๖ | มาตรากฎหมาย |
| `ออกหมาย ...` | แบบ ๖ | หมายเรียก/จับ |
| `เหตุผล ...` | แบบ ๕๗ | เหตุผลขอปล่อยชั่วคราว |
| `หลักประกัน ...` | แบบ ๕๗ | ประเภทหลักประกัน |
| ไม่มี continuous_text | แบบ ๔, ๓๒, ๓๖ | เนื้อหาคำฟ้อง/อุทธรณ์/ฎีกา |

## 4. เกณฑ์ตรวจบัญชีพยาน

| เกณฑ์ | คดีแพ่ง | คดีอาญา |
|-------|--------|---------|
| พยานเอกสารขั้นต่ำ | ≥ 1 (สัญญา/หลักฐาน) | ≥ 1 |
| พยานบุคคลขั้นต่ำ | ≥ 1 (โจทก์) | ≥ 1 (ผู้เสียหาย) |
| facts_to_prove | ต้องระบุทุกรายการ | ต้องระบุทุกรายการ |
| ลำดับ (sequence) | ต้องเรียงต่อเนื่อง | ต้องเรียงต่อเนื่อง |

### พยานเอกสารที่ควรมี (ตามประเภทคดี)

| ประเภทคดี | พยานเอกสารสำคัญ |
|-----------|---------------|
| กู้ยืมเงิน | สัญญากู้ยืม, หนังสือทวงถาม |
| ซื้อขาย | สัญญาซื้อขาย, ใบเสร็จ, โฉนด |
| ละเมิด | ภาพถ่าย, ใบรับรองแพทย์, บันทึกประจำวัน |
| ฉ้อโกง | สัญญา, ใบเสร็จ, หลักฐานการโอนเงิน |
| เช็ค | เช็คคืน, หนังสือแจ้งเช็คคืน |
| แรงงาน | สัญญาจ้าง, ใบจ่ายเงินเดือน, หนังสือเลิกจ้าง |

## 6. เกณฑ์ตรวจเลขไทย

| เกณฑ์ | ถูก | ผิด |
|-------|-----|-----|
| ตัวเลขไทย | U+0E50-0E59 (๐๑๒๓๔๕๖๗๘๙) | U+0E60-0E69 (๠๡๢...) |
| พ.ศ. | ค.ศ. + 543 (เช่น 2026 → ๒๕๖๙) | ค.ศ. ตรงๆ |
| จำนวนเงิน | ตัวเลขไทย + ตัวอักษร | ตัวเลขอารบิกใน body |
| เลขคดี | ตัวเลขไทย /๒๕xx | ตัวเลขอารบิก |

### วิธีตรวจเลขไทยผิด range

ค้นหา character U+0E60-0E69 ใน body_html และ continuous_text:
```python
# ถ้าเจอ character ใน range U+0E60-0E69 = ผิด
any(0x0E60 <= ord(ch) <= 0x0E69 for ch in text)
```
